Conceptos base
¿Qué es una BAPI y por qué existe?
Antes de ejecutar nada, entendemos el concepto. Una BAPI es la base de casi todo desarrollo de integración en SAP.
SAP Standard Módulo PP Sin código requerido
Una BAPI es como una ventanilla de banco. Tú no entras a la sala de servidores a mover el dinero tú mismo — le das los datos a la ventanilla, ella valida, procesa y te devuelve la respuesta. SAP ya programó esa ventanilla. Nosotros solo la usamos.
— Analogía para equipos funcionales
Definición técnica

BAPI (Business Application Programming Interface) es un módulo de función SAP estándar, con una interfaz estable, que permite crear, modificar o consultar objetos de negocio (órdenes, materiales, proveedores, etc.) de forma programática y segura.

💡
A diferencia de otros módulos de función, las BAPIs son estables entre versiones de SAP. Si un programa las usa hoy, seguirá funcionando en una versión futura del sistema.

Las tres pestañas que debes conocer

Lo que nosotros metemos a la función. Son los datos de entrada que necesita para ejecutar su lógica.

📥
Aquí van el material, la planta, el tipo de orden, la cantidad, las fechas. Son los parámetros obligatorios para que la BAPI sepa qué crear.

Lo que la función nos devuelve si la ejecución fue exitosa.

📤
El campo más importante aquí es ORDER_NUMBER — el número de la orden recién creada. Ojo: este campo solo tiene valor si no hubo errores. Para saber si hubo errores, siempre hay que revisar primero la pestaña TABLES.

La tabla RETURN es la más importante. Es donde SAP nos dice si todo fue bien o si hubo algún problema.

⚠️
Regla de oro: siempre revisa TABLES → RETURN primero. El campo TYPE te dice el veredicto: S = éxito, E = error, W = advertencia. Solo si es S vas a EXPORT a buscar el ORDER_NUMBER.

Buenas prácticas
Consultar antes de crear
Un paso que la mayoría salta y que evita el 80% de los errores. Antes de crear, consultamos una orden existente para ver exactamente cómo vienen los datos.
SE37 BAPI_PRODORD_GETDETAIL Lectura · Sin cambios
Por qué usar GetDetail primero

Los campos en SAP tienen formatos específicos — un material puede tener ceros a la izquierda, una planta puede ir con o sin punto, etc. Si los mandamos en el formato incorrecto, la BAPI devuelve error.

✨
La estrategia: tomar una orden de producción que ya existe en el sistema, consultarla con GetDetail, ver cómo SAP tiene guardados esos valores y usar exactamente esos mismos formatos en el Create.
Abrir dos sesiones en paralelo

Para trabajar cómodamente vas a necesitar dos ventanas de SE37 abiertas al mismo tiempo — una con el GetDetail y otra con el Create. Así puedes consultar los datos en una y copiarlos en la otra sin perder nada.

SAP · Cómo abrir sesión paralela 1. En SAP, menú superior → Sistema → Crear sesión
O también puedes usar el atajo /o en la barra de transacciones
2. Se abre una nueva ventana de SAP independiente
3. En esa nueva ventana abre SE37 con BAPI_PRODORD_GETDETAIL
4. En la ventana original deja BAPI_PRODORD_CREATE lista para rellenar
✨
Con ambas ventanas lado a lado puedes ver exactamente cómo SAP tiene guardado cada valor en el GetDetail y copiarlo directamente al Create — sin adivinar formatos ni cometer errores.
Cómo usar GetDetail en SE37
Ventana 1 — GetDetail 1. Buscar: BAPI_PRODORD_GETDETAIL
2. En IMPORT → NUMBER = número de orden existente (ej: 000001234567)
3. En IMPORT → ORDER_OBJECTS → marcar HEADER = X como mínimo
4. Ejecutar con F8
5. Revisar TABLES → ORDER_OBJECTS para ver los campos con sus valores reales
6. Anotar: MATERIAL, PLANT, ORDER_TYPE, QUANTITY y formato de fechas

Ejecución y resultado
Ejecutar y leer el resultado
Presionamos F8 y SAP nos devuelve una respuesta. Saber leer esa respuesta es tan importante como saber ejecutar la BAPI.
¿Qué es la secuencia de test y por qué usarla?

La secuencia de test es una función de SE37 que te permite encadenar varias BAPIs y ejecutarlas en orden con una sola acción. En nuestro caso necesitamos ejecutar dos: primero el Create y luego el Commit. Sin esta herramienta tendrías que hacerlo en dos sesiones separadas y manualmente, con riesgo de olvidar el Commit.

✨
La secuencia de test garantiza que ambas BAPIs se ejecuten en el orden correcto y dentro de la misma sesión. Es la forma estándar de probar BAPIs en SE37 cuando se necesita más de una llamada.
Cómo llegar a la secuencia de test
SE37 · Navegación 1. Abrir SE37 y buscar BAPI_PRODORD_CREATE
2. En el menú superior ir a Módulo de funciones → Ejecutar → Secuencia de test
3. Se abre la pantalla de secuencia — aquí es donde añadiremos las dos BAPIs
Añadir las BAPIs en la secuencia
Secuencia de test · Paso 1 — Agregar Create 1. En la pantalla de secuencia, toca Insertar módulo de funciones (o el ícono +)
2. Escribe BAPI_PRODORD_CREATE y confirma
3. Se añade como primer paso de la secuencia
4. Selecciónala y rellena los campos IMPORT con los valores del Paso 4
(MATERIAL, PLANT, ORDER_TYPE, QUANTITY, fechas)
Secuencia de test · Paso 2 — Agregar Commit 5. Vuelve a tocar Insertar módulo de funciones
6. Escribe BAPI_TRANSACTION_COMMIT y confirma
7. Se añade como segundo paso — justo después del Create
8. Selecciónala y en el campo WAIT introduce X
Secuencia de test · Ejecutar 9. Con ambas BAPIs en la secuencia, ejecuta con F8
10. SAP ejecuta primero el Create y luego el Commit automáticamente
11. Revisa el resultado del Create → TABLES → RETURN → campo TYPE
12. Si TYPE = S → ve a EXPORT → ORDER_NUMBER y anota el número de orden
¿Por qué existe el COMMIT? — La analogía del carrito
Imagina que estás comprando en un supermercado online. Vas añadiendo productos al carrito — pero eso no significa que hayas comprado nada. El carrito es temporal. Solo cuando pulsas "Confirmar pedido" la compra se hace real.

SAP funciona igual. La BAPI añade la orden al "carrito" de la base de datos. El COMMIT es el botón de "Confirmar". Sin él, SAP descarta todo al cerrar la sesión.
— Por eso siempre van juntos en la secuencia
Interpretar la tabla RETURN

El campo TYPE te dice todo lo que necesitas saber:

S Success
Orden creada y grabada.
E Error
Falló. Ver MESSAGE y corregir.
W Warning
Creó con advertencia. Revisar.

Verificar la orden en CO03
El paso que conecta todo. Aquí ves que lo que hiciste en SE37 — o lo que haría un programa — tiene un resultado real y concreto en SAP.
CO03 Solo lectura · Sin riesgo
Pasos para verificar
SAP 1. Ir a transacción CO03
2. Introducir el número de orden devuelto por la BAPI
3. Ejecutar con Intro o F8
4. Verificar: material, cantidad, fechas, estado
Lo que acabas de hacer manualmente en SE37, un programa ABAP lo puede ejecutar automáticamente con los mismos campos y la misma lógica — para 1 orden o para 10.000. La diferencia no es qué hace, sino quién lo hace y cuántas veces.
